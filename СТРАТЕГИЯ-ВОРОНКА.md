# Стратегия автоворонки — ViboClub / Matryca Losu

Рынок: PL. Канал входа: Instagram @matryca_astro_kate (81k).
**Без email.** Длинный прогрев: Telegram (открытия 50–90%) + ManyChat (директ, окно 24ч продлеваем интерактивом) + ретаргет через Meta Pixel.

---

## 1. Архитектура связки (кто за что отвечает)

```
INSTAGRAM            → охват, первое касание по боли (рилс/сторис/реклама)
   │
MANYCHAT (IG DM)     → ловит по триггеру, отдаёт лид-магнит, ЗАБИРАЕТ В TELEGRAM
   │
TELEGRAM-БОТ         → длинный прогрев (тут живёт прогрев, а не в почте)
   │
САЙТ (matryca)       → «память» воронки: метит, кто что сделал, шлёт события
   │
META PIXEL/CAPI      → собирает аудитории для ретаргета и lookalike
```

Правило: **ManyChat греет первые сутки и уводит в Telegram. Telegram держит длину. Сайт всё фиксирует и докладывает обратно тегами.**

## 4. Этапы воронки — детально (ТЗ)

### ЭТАП 0 — Трафик / Вход
- **Осведомлённость ЦА (Шварц/Хант):** Unaware → Problem-aware. Человек чувствует «вроде всё ок, но что-то не работает», ещё не знает про Матрицу как решение.
- **Цель:** клик/реакция → перевод в ManyChat.
- **Канал:** IG рилс, сторис, таргет.
- **Контент (без копирайтинга):** хук по конкретной боли — деньги/отношения/«хожу по кругу/повторяющийся сценарий». НЕ «рассчитай матрицу» (этого слова она ещё не ищет), а через симптом. CTA: «напиши слово в директ».
- **Триггер:** одно слово-захват (напр. `MATRYCA` / `START`). Максимум 2–3 активных.
- **Условие перехода:** комментарий/DM с триггером → автозапуск ManyChat.

### ЭТАП 1 — Захват лида (ManyChat, IG DM)
- **Осведомлённость:** Problem-aware. Боль названа, решение ещё не выбрано.
- **Цель:** отдать лид-магнит + завести в Telegram + забрать в базу.
- **Логика ManyChat (первые 24ч, продлеваем интерактивом):**
  1. Приветствие + сразу польза (мини-инсайт по боли из хука).
  2. Ссылка на бесплатный расчёт на сайте с зашитым ID: `?mc={{user_id}}`.
  3. Кнопка «получить полный разбор бесплатно в Telegram» → ведёт в бот (тут сделка: длинный прогрев переезжает в TG, где нет окна 24ч).
  4. Каждое сообщение заканчивается кнопкой/вопросом → окно 24ч переоткрывается.
- **Сегментация на входе:** тег по теме хука (деньги / отношения / предназначение) — дальше прогрев под этот сегмент.
- **События наружу:** сайт по `?mc` ставит тег `calc_done`, как только человек получил расчёт.

### ЭТАП 2 — Прогрев (Telegram, 5–7 дней)
- **Осведомлённость:** Solution-aware. Знает, что Матрица = решение, выбирает, доверять ли тебе.
- **Цель:** доверие + снять возражения + подвести к tripwire.
- **Канал:** Telegram-бот (драип-сообщения) + параллельно прогревочные сторис в IG.
- **Смысловая сетка (порядок не менять):**

| День | Смысл | Контент (без копирайтинга) |
|---|---|---|
| 1 | Мечта | Какой результат даёт понимание своей матрицы (картинка «как может быть») |
| 2 | Проблема | Почему ты ходишь по кругу — и это НЕ твоя вина (механизм) |
| 3 | Решение/авторитет | Метод Ладини + твой опыт 2500 разборов, почему не гадание |
| 4 | Соц. доказательство | Конкретный кейс/отзыв под сегмент (деньги/отношения) |
| 5 | Возражения (FAQ) | «это эзотерика?», «а точно про меня?», «чем отличается от гороскопов» |
| 6 | Тизер + оффер | Открываешь tripwire (дешёвый PDF) |
| 7 | Дедлайн | Напоминание + срок/бонус, последний призыв |

- **Фишки:** петля «продолжение завтра», вопросы аудитории (отвечают = вовлечены = окно открыто), счётчик мест/срока.
- **События наружу:** клик на оффер из TG → сайт метит `viewed_pricing`.

### ЭТАП 3 — Tripwire (EVERGREEN, всегда включён)
- **Осведомлённость:** Product-aware. Знает твой продукт, сомневается в цене/ценности.
- **Цель:** первая транзакция (превратить подписчика в покупателя — дальше продавать в разы легче).
- **Продукт:** недорогой PDF-разбор через Stripe (уже стоит).
- **Контент:** оффер с конкретикой, ясная маленькая цена, мгновенная отдача.
- **События:** `InitiateCheckout` при старте оплаты, `Purchase` (value+currency) при оплате → тег `tripwire_buyer`.
- **Развилка:** не купил за X дней → ретаргет-реклама + догрев в TG. Купил → Этап 4.

### ЭТАП 4 — Core offer (LAUNCH, по расписанию)
- **Осведомлённость:** Most aware. Уже купил мелочь, доверяет, готов к большому.
- **Цель:** продать полный разбор / консультацию.
- **Механика LAUNCH:** периодические открытия с дедлайном и ограничением мест (живая консультация = ограниченный ресурс). Прогрев перед открытием — серия в TG + сторис + эфир/вебинар в конце (макс. конверсия).
- **Контент:** глубокий кейс-разбор, эфир «разберу вживую 3 матрицы», оффер + дедлайн + бонус.
- **События:** `Purchase` (core, высокий value) → тег `core_buyer`.

### ЭТАП 5 — Maximizer / удержание (LTV)
- **Осведомлённость:** Customer. Лоялен.
- **Цель:** подписка ViboClub (return-product) + допродажи.
- **Контент:** онбординг в приложение, регулярная ценность (медитации/практики под результат матрицы), upsell на сопутствующее, downsell для не купивших core.
- **События:** `Subscribe` → тег `app_sub`.

---

## 5. LAUNCH + EVERGREEN вместе

- **EVERGREEN (фон, 24/7):** Этапы 0→1→2→3. Любой новый человек автоматически идёт лид-магнит → прогрев → tripwire. Работает без тебя.
- **LAUNCH (волны):** Этап 4 (core) и Этап 5 (подписка) запускаешь периодически — открытие консультаций / запуск когорты с дедлайном. Прогрев к лончу берёт **тёплую базу**, которую уже накопил evergreen (все, у кого тег `tripwire_buyer` и `calc_done`, но нет `core_buyer`).

То есть evergreen копит и греет базу, а launch её монетизирует волнами.

## 6. ТЗ трафик-менеджеру (события + аудитории)

### События передавать (Pixel + Conversions API, на каждом этапе):

| Этап | Событие | Когда фаerr | Параметры |
|---|---|---|---|
| 0 | `PageView` | вход на сайт | авто |
| 0–2 | `ViewContent` | открыл статью/прайс | content_name |
| 1 | `Lead` | получил расчёт матрицы | + tg-переход |
| 2 | `tg_joined` (custom) | вошёл в Telegram-бот | sub_id |
| 3 | `InitiateCheckout` | начал оплату | value, currency |
| 3 | `Purchase` (tripwire) | оплатил PDF | value, currency |
| 4 | `Purchase` (core) | оплатил разбор | value, currency |
| 5 | `Subscribe` | оформил подписку | predicted_ltv |

### Аудитории — СБОР:
- **A1 (холодные→тёплые):** все, кто взаимодействовал с IG-аккаунтом (180 дн) + все посетители сайта (180 дн). → для ретаргета на Этап 0/1.
- **A2 (главный тёплый пул):** `Lead` (расчёт сделан), НЕ `Purchase`. → ретаргет на tripwire.
- **A3 (брошенная корзина):** `InitiateCheckout` без `Purchase` за 7 дн. → самый горячий ретаргет.
- **A4 (tripwire-покупатели):** `Purchase` tripwire. → аудитория для LAUNCH core.
- **A5 (core-покупатели):** → для оффера подписки ViboClub.
- **LAL:** lookalike 1–3% от `Purchase`-аудиторий (на холодный трафик Этапа 0).

### Аудитории — ИСКЛЮЧЕНИЯ (обязательно):
- Реклама tripwire → **исключить** A4 и A5 (уже купили — не жги бюджет).
- Реклама core (LAUNCH) → **исключить** A5 (core-покупателей).
- Реклама лид-магнита → **исключить** всех, у кого уже `Lead`.
- Подписка ViboClub → **исключить** `app_sub`.

Правило: на каждом шаге всегда исключаем тех, кто этот шаг уже прошёл.

## 7. Метрики (контроль по этапам)

| Этап | Метрика | Ориентир |
|---|---|---|
| Вход | CTR хук → триггер | ≥1–3% |
| Захват | дошли до расчёта | ≥80% от вошедших в ManyChat |
| Перевод в TG | % забравшихся в Telegram | следить, это ключевой мост |
| Прогрев | досмотр день1→день7 | чинить провалы по дням |
| Tripwire | конверсия в покупку | ≥10–20% от дошедших до оффера |
| Core | конверсия LAUNCH | ≥20–30% от тёплой базы |
| LTV | % взявших подписку | ключ для возврата на рекламу |

Падает этап — чинишь его, не всю воронку: слабый вход = хук; мало доходят до лид-магнита = ManyChat/CTA; не переходят в TG = оффер перехода; не покупают = прогрев/доверие.
